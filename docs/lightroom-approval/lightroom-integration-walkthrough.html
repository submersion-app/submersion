Submersion + Adobe Lightroom

Submersion is a scuba dive-logging app. This integration links a diver's Adobe Lightroom cloud photos and videos to their dive logs automatically, by capture time.

What it does

After a diver connects their Adobe account, Submersion reads their Lightroom cloud catalog, matches photos and videos to dives by the moment they were captured, and shows each shot on the corresponding dive — with a one-tap "Open in Lightroom" link back to the original. It is read-only: Submersion never modifies the Lightroom catalog.

How a user connects (end to end)

  1. Open Submersion → Settings → Photos & Media → Connect Lightroom.
  2. The system browser opens Adobe's sign-in page; the user signs in with their Adobe ID and approves access.
  3. The browser returns to Submersion and the account is connected. No developer setup, no keys — the user only signs in.
  4. Submersion scans the diver's trips and attaches confident photo matches; ambiguous ones appear as one-tap suggestions on the dive.
  5. Each linked photo shows on its dive, with "Open in Lightroom" to view the original.

What it accesses, and why

ScopeWhy Submersion requests it
openid, AdobeIDIdentify the signed-in Adobe user (account name / email shown in Settings).
lr_partner_apisRead the user's Lightroom catalog: account, catalog id, albums, and asset metadata (capture time, filename) for matching.
lr_partner_rendition_apisFetch a display rendition of each matched photo/video to show it in the app.
offline_accessRequested so the connection can persist; used only if Adobe issues a refresh token for the credential.

Technical summary (for the review team)

Integration typeAdobe Lightroom Services API (Creative Cloud) via Adobe IMS OAuth 2.0. Not the Adobe Express Embed SDK.
CredentialOAuth Native App (public client, PKCE, no client secret). Client ID: [CLIENT ID].
AuthorizationThree-legged, user-consented. Authorize: ims-na1.adobelogin.com/ims/authorize/v2; token: ims-na1.adobelogin.com/ims/token/v3.
API baselr.adobe.io — /v2/account, /v2/catalog, /v2/catalogs/{id}/albums, /v2/catalogs/{id}/assets, /v2/catalogs/{id}/assets/{id}/renditions/{size}.
PlatformNative app (Flutter) for macOS, Windows, Linux, iOS, and Android — which is why the workflow is shown here and in the demo video rather than as a hosted web app.
Data handlingRead-only. Renditions are cached on the user's device / the user's own cloud store. Submersion operates no server that stores Adobe tokens or photos.
