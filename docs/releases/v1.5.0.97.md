# 🎉 Submersion v1.5.0.97 Release Notes

This is a major release, large enough to move the version to 1.5. The centerpiece is **multi-device cloud sync**, arriving as a fully supported feature for the first time. Submersion can now keep a dive log in step across all of a diver's devices using **iCloud** on Apple platforms, or any **S3-compatible storage** (AWS S3, Cloudflare R2, Backblaze B2, MinIO, or a self-hosted server) on every platform. There is no Submersion server in the middle and no account to create: the data lives in the user's own iCloud account or storage bucket.

For setup and an explanation of how sync works, see the Multi-Device Sync guide: [github.com/submersion-app/submersion/wiki/Multi-Device-Sync](https://github.com/submersion-app/submersion/wiki/Multi-Device-Sync)

This release also rebuilds the dive edit screen, overhauls the efficiency of the sync engine, makes restoring a backup considerably safer, and adds the ability to record diving done before the app was first used so that lifetime totals are accurate. A round of import and dive-computer fixes is included as well.

---

## ✨ New features

### Multi-device cloud sync (iCloud and S3-compatible storage)

On Apple platforms, **iCloud** is a fully supported sync backend with no configuration: the dive log syncs through the diver's own iCloud account.

On every platform, Settings also includes an **S3 storage** provider that can be pointed at any S3-compatible endpoint. Setup takes an endpoint URL, a bucket, and access keys. Credentials are stored in the device keychain rather than in the database or a plain file.

Details worth noting:

- **Provider-neutral** keeps the form clear across AWS, Cloudflare R2, Backblaze B2, and self-hosted MinIO.
- **Credentials are retained on sign-out** so a device can reconnect, while the synced library metadata is cleared cleanly.

The Multi-Device Sync guide (linked above) includes a **Cloudflare R2** walkthrough. R2's free tier is sufficient to sync a dive log at no cost, which makes it a practical option for divers who do not already have S3 storage.

More backends are on the way. Additional storage providers are planned for upcoming releases. For divers syncing across a mix of Apple and non-Apple devices today, the S3-compatible backend is the one that reaches all of them.

### Redesigned dive edit form

The dive editor has been rebuilt on a shared set of form components:

- **Hero stats** for depth, duration, and temperature, editable in place.
- **Collapsible sections** (The Dive, Gas & Gear, Conditions, Trip, Buddies, Experience) that summarize when closed, keeping a long form scannable.
- **Tank cards** with an inline, expanding editor for each cylinder.
- **A two-column layout on wide windows** for desktop and tablet, in place of a single long column.
- **A discard-confirmation guard** to prevent losing edits when backing out, and **automatic expansion of any section containing a validation error**.

The dive-site edit screen was rebuilt on the same foundation, including a merge mode, so the two screens are now consistent.

The dive edit form will continue to be enhanced iteratively across upcoming releases, with further refinements building on this new foundation.

### Prior dive experience and career totals

Statistics previously began counting from the first dive logged in the app, which understated the lifetime numbers for experienced divers. Each diver profile can now record a **prior dive count, prior time underwater, and a "diving since" date**. The Statistics screen folds these figures into lifetime totals and shows a **logged plus prior breakdown**, so career numbers reflect a diver's full history.

### Safer backup restore

Two rough edges around restoring a backup have been addressed:

- **Restore now asks how it should be applied:** *Merge* into the data already on the device, or *Replace everything*.
- **Cloud sync resumes automatically after a restore.** The app re-establishes sync on the next launch on its own, and when a replaced library is detected, other devices surface it and offer to adopt it rather than competing over which copy is authoritative.

## ⚡ Improvements

### Faster, lighter sync

Sync no longer re-uploads the entire database whenever something changes. It now publishes only what changed since the previous sync (a per-device changeset log on top of a compacted base snapshot), verifies everything against checksums on download, and compacts itself over time. For large logs or several devices, this means less data transferred and quicker syncs, with the same outcome: every device converges on the same library.

### Safe backend switching

Moving from one storage backend to another is now handled deliberately, with guards against data loss and "split-brain" situations where two devices each treat themselves as the source of truth.

### Sync upgrade-path hardening

Several robustness improvements for divers enabling sync or upgrading: first-contact library merges now confirm before combining two libraries, duplicate "twin" device identities are detected to avoid redundant uploads, resetting sync state preserves deletion history so removed dives do not reappear, and a missing or unreadable cloud library recovers rather than halting sync.

## 🐞 Bug fixes

- **Nitrox dives importing as air.** When a dive computer reports gas mixes but no separate tank records (common on computers without air integration), the dive was logged on plain air. The gas is now taken from the reported mixes, so nitrox dives import as nitrox.
- **Aqualung i330R download timeouts.** Bluetooth reads occasionally merged two notifications into one and corrupted the stream. Notification boundaries are now preserved, resolving stalled and timed-out downloads on the i330R and helping other BLE computers.
- **Dive computer TTS not displayed.** Per-sample time-to-surface values reported by the computer are now persisted, so the dive computer TTS source has data to display.
- **Database recovery.** Databases stranded by an earlier internal schema-version collision are detected and repaired on launch, recovering sync metadata that had been left behind.
- Numerous smaller sync correctness and resilience fixes, including cold-starting a fresh base when the cloud copy is missing, retaining the upload nonce through a timeout, and checksum verification on read.
