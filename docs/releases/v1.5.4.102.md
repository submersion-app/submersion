# 🛠️ Submersion v1.5.4.102 Release Notes

v1.5.4 is a stability and correctness release that hardens the v1.5 cloud-sync rollout across every platform. The headline items are a **sync data-loss fix** for equipment sets and per-dive gear, a **Windows fix** that restores cloud sync and map tiles, and a **backup location** that could lock a diver out of the app at startup. There are also fixes for the macOS direct-download (DMG) build, more accurate air-consumption (SAC) figures on multi-tank dives, and a couple of UI papercuts.

If you sync across two or more devices, this is an important update.

---

## ✨ New and improved

### Reliable custom backup folders on iPhone, iPad, and Mac

You can now choose a custom backup folder on Apple platforms and have it keep working across app launches. Previously a custom location (for example, an iCloud Drive folder) could quietly lose access between launches; the app now holds a secure reference to the folder you pick, the same way the custom database-location feature already did. (See the related fix below for what used to happen when that access was lost.)

### Prior dive experience is now reachable

v1.5 introduced prior-experience fields — a prior dive count, prior time underwater, and a "diving since" date that fold into your lifetime statistics. Those fields now have a home in the new **Diver Profile** screen, so you can actually enter and edit them. Clearing a field (emptying a blood type, removing a medical-clearance date, or zeroing a prior dive count) now saves correctly as well.

## 🐞 Bug fixes

### Gear sets and per-dive gear lost when syncing

The most important fix in this release. When syncing across devices, an **equipment set** kept its name but arrived on the other device with **all of its gear dropped**, and the gear attached to a **dive** could vanish on edit. The cause was in how those membership lists are saved and then merged during sync. Membership for equipment sets and per-dive gear is now preserved across devices, and the next sync repairs any device that was already affected.

### Cloud sync and map tiles failed on Windows

On Windows, every secure connection the app made — S3/R2 cloud sync, map tiles, photos loaded from the web, weather, and reverse geocoding — failed with a certificate error, even though the same sites loaded fine in a browser on the same machine. Submersion now reads the **Windows certificate store** (with a bundled fallback), so cloud sync and maps work on Windows. macOS, iOS, Android, and Linux were not affected.

### A custom backup location could lock you out of the app

If your backup folder pointed somewhere that had become inaccessible (commonly an iCloud Drive folder), the safety backup the app takes before a database upgrade could fail on startup with only **Retry** and **Quit** — and Retry just ran the same failing step again, leaving no way to reach Settings to fix it. Startup is no longer blocked by an unusable backup location, and the reliable custom-folder support above prevents that access from being lost in the first place.

### macOS direct-download (DMG) build: launch crash and broken cloud credentials

The DMG distributed directly from GitHub could **fail to launch at all** on macOS for one build, and once that was resolved, **S3 credentials would not save** so cloud sync failed with a keychain error. Both problems came from keychain entitlements that the direct-download build is not allowed to use. The DMG build now launches and stores S3 credentials correctly. App Store and TestFlight builds were never affected.

### iCloud offered on Mac builds that can't use it

The macOS direct-download build cannot use iCloud — Apple restricts iCloud to App Store and development-signed builds — but the iCloud sync option was still shown and then failed with a misleading "please sign in to iCloud" message even when you were already signed in. The app now checks whether the running build can actually use iCloud and only offers it when it can, with a clear explanation otherwise. On those builds, **S3-compatible storage** (AWS S3, Cloudflare R2, Backblaze B2, MinIO, or self-hosted) remains the path to sync.

### Air consumption (SAC) wrong on multi-tank dives

On dives with more than one tank, the surface-air-consumption figure was calculated from only the first tank — both in the Dives list and on the Statistics charts and best/worst records. SAC now accounts for all of the gas used on a dive (pressure-based SAC uses the back-gas tank, since adding up pressure drops across cylinders of different sizes is not meaningful). The Dives-list **SAC column** now also respects your configured **SAC unit**, so imperial divers see cuft/min or psi/min instead of a metric value with the wrong label.

### Save button invisible with some themes on iPhone and iPad

On iOS, with the Tropical theme (and possibly others), the **Save** button on the dive detail screen was the same color as the header and effectively invisible — tapping where it should be still worked, but you couldn't see it. The button text is now legible across themes.

### Smaller fixes

- Map tile loading failures are now logged instead of being silently swallowed, which will make any future map issues easier to diagnose.
