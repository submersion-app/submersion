# 🎉 Submersion v1.5.6.108 Release Notes

v1.5.6 is a fast follow-up to v1.5.5, led by the single most-requested community feature: editing many dives at once. It also turns Garmin FIT import into a full-fidelity importer (air integration, gas, deco, GPS, time zone), finally makes the dive-profile Ascent Rate toggle actually draw a curve, and clears a batch of reported bugs - most importantly a download crash on the newest Android phones, plus fixes for disappearing SAC graphs, reverted dive-site locations, and a stale Home tab after import or sync.

---

## ✨ New and improved

### Bulk dive editing

You can now select multiple dives and edit them all in one pass. Multi-select a set of dives, open the new bulk-edit form, and change any of the fields you tick — the rest are left untouched.

- **Stamp shared details across a whole trip or boat day:** dive center / operator, conditions (water and air temp, visibility, current, swell, entry/exit, altitude, surface pressure), weather (wind, cloud, precipitation, humidity), and logistics fields. 27 individual fields can be set in bulk, each with its own on/off toggle so you only change what you intend to.
- **Add, remove, or replace collections in bulk:** tags, equipment, buddies, weights, tanks, and marine-life sightings — each with a clear **Add / Remove / Replace** mode (for example, add a buddy to every selected dive, or replace the tank on a batch of dives that imported without one).
- **Notes** can be appended to every selected dive without overwriting what's already there.
- **One-tap undo:** every bulk change is reversible per dive, so an accidental edit is easy to back out.
- **Faster selecting:** shift-click to select a range, select by date range, and Select All / Deselect All on both phone and tablet/desktop layouts.

The bulk form is the same dive editor you already know, running in a bulk mode, so it looks and behaves consistently. Under the hood it applies one set-based database update instead of editing dives one by one, so it stays fast even on large batches.

### Comprehensive Garmin FIT import

Importing a Garmin `.fit` file now brings in everything the file contains, not just depth, temperature, and heart rate:

- **Air integration** — tank start/end pressures and the full per-sample pressure series from a T2 transmitter, including multi-transmitter setups. This feeds your SAC rate.
- **Gas mixes**, including nitrox and multi-gas dives.
- **Decompression data** recorded by the computer.
- **Entry and exit GPS**, not just the entry point.
- **Correct local time** resolved from the file's own time zone, so dives land at the right wall-clock time.
- **Proper model names** for Garmin Descent computers instead of a generic label.

This was validated end-to-end against real Descent Mk3i, Descent X50i, and T2-transmitter files from divers who tested it.

### Ascent rate now actually graphs

The **Ascent Rate** toggle on the dive profile previously only added a row to the hover tooltip — it never drew anything on the chart, so turning it on appeared to do nothing. It now draws your ascent rate on the profile (velocity coloring plus a dedicated ascent-rate line), and works for imported dives too since the rate is derived from the depth/time samples.

Alongside that, the two ascent-rate controls were untangled and given matching names in the chart and in Settings, both now **default off**, and the separate "Ascent Rate Line" is now a real, saved setting rather than a session-only toggle.

### Two-finger trackpad scroll-to-zoom on maps and the profile chart

On a trackpad, a two-finger scroll (and pinch) now zooms in and out, anchored to where your cursor is pointing, across all 17 maps in the app and the dive profile chart. Panning stays on click-drag. This also fixes a long-standing annoyance: scrolling over a map embedded in a scrollable page now zooms the map instead of scrolling the page past it.

### "Calculate average depth" button is back

The one-tap button that calculates average depth from your dive profile had been buried by an earlier redesign into a faint, easy-to-miss icon that took two taps. It's restored as an obvious, single-tap affordance on the dive edit form, and the top of the edit screen ("The Dive") was flattened so dive number, entry, and exit are right at the top.

## 🐞 Bug fixes

### Download crash on the newest Android phones (16 KB memory pages)

On Android 15 devices that use 16 KB memory pages (such as the Xiaomi 15T Pro), starting a dive-computer download, reported with a Mares Puck Pro, could shut the app down. The cause was the bundled native dive-computer library being aligned for the older 4 KB page size, which the system refuses to load on these devices. The library is now correctly 16 KB-aligned, and if any native library ever fails to load again, the app now fails gracefully with a clear message and logs the error instead of crashing silently. A build-time check guards against the misalignment ever shipping again.

### OSTC nano Bluetooth downloads complete reliably

OSTC nano computers (and the wider hw_ostc3 family) could connect over Bluetooth and start downloading but intermittently stop partway with an error. Downloads now retry and keep the Bluetooth notification queue unblocked, so transfers that previously stalled run to completion. (This is the iOS-side remainder of the OSTC nano fix that began in v1.5.5.)

### SAC graph no longer disappears after editing a dive

After editing a dive's site, trip, or operator, the SAC rate graph could vanish from the profile panel and SAC could no longer be picked from the metric dropdown — even though nothing about the gas or tank data was changed. The profile now keeps your SAC curve in that situation.

### Dive-site Country and Region edits now stick

When editing a dive site, manually changing or clearing the **Country** or **Region** was silently overwritten on save by an automatic reverse-geocode lookup (for example a Grand Turk site's Region reverting to "St. Croix", or Bonaire's Country flipping to "Caribbean Netherlands"). Your manual edits are now permanent, while automatic suggestions still fill in fields you leave untouched.

### Home tab refreshes after an import or sync

After downloading dives from a dive computer or syncing from iCloud, the Home tab could keep showing stale recent dives and headline statistics until you restarted the app, even though the dive list itself updated. The Home tab now refreshes its recent dives and stats when the database is written directly.

### Large-library sync memory fix, now on the "Replace" path too

v1.5.5 streamed sync data to avoid running out of memory on large libraries, but it only converted the normal startup adoption path. Devices adopting a library that had been **replaced** (an epoch change) were still loading the entire library into memory and could crash on launch on iOS. That path now streams the same way, so adopting a replaced library uses a small, bounded amount of memory.
