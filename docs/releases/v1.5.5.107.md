# 🎉 Submersion v1.5.5.107 Release Notes

v1.5.5 is a feature release focused on getting your dives off your dive computer and into Submersion accurately. It fixes the USB-serial download path, fixes identification and download for several Bluetooth models, brings proper rebreather (CCR/SCR) handling to imports and oxygen-exposure math, and makes the dive profile chart far nicer to zoom and pan on every kind of device. Dive sites gain City / Island / Body of Water fields, Android gets reliable backup folders, and large-library sync no longer runs out of memory on memory-constrained devices.

---

## ✨ New and improved

### USB-serial dive computer downloads (macOS, Windows, Linux, and Android)

Dive computers that connect over a serial-over-USB cable, such as the Mares Puck Pro and similar models, can now be downloaded directly. Previously these devices would connect but fail partway through the transfer. On Android this works through the USB Host API (plug in via an OTG adapter); on the desktop it uses the platform serial port. If a cable is unplugged or USB permission is revoked mid-download, the app now reports a clear I/O error and stops, instead of silently hanging until it times out.

### Rebreather (CCR / SCR) support in imports and oxygen math

Closed-circuit and semi-closed-circuit dives are now handled correctly end to end:

- **All O2 sensors are imported**, not just the first cell, so divers with redundant oxygen sensors keep their full sensor data.
- **CNS and OTU oxygen-exposure figures are now computed from the loop ppO2 you actually breathed** (your computer's ppO2, the average of your O2 cells, or your setpoint), rather than the open-circuit "depth × FO2" estimate. That open-circuit fallback was badly overstating CNS on rebreather dives. Max ppO2 and ppO2 warning events now come from the same curve drawn on your profile chart, so the numbers agree.

### Smarter, smoother dive profile chart

The dive profile chart got a navigation overhaul that adapts to whatever you're using:

- **Zoom is anchored to your cursor or pinch point** — the part of the dive you're pointing at stays put as you zoom in, instead of the chart jumping around.
- **Desktop:** hover to read values, and **click-drag to pan**.
- **Trackpad:** **pinch to zoom toward the cursor** and **two-finger drag to pan**.
- **Touch:** **double-tap-and-hold, then drag to pan**.
- Fixed a macOS bug where a trackpad pinch or two-finger pan would jump to the lower-right corner.

### Dive site location detail: City, Island, and Body of Water

Dive sites can now record a **City**, **Island**, and **Body of Water** in addition to the existing fields. These appear on the site edit form and detail view, and feed into site search and the combined location string.

### Reliable backup folders and storage choice on Android

Android backups now use a proper folder picker (the system document picker), so you can back up to a location of your choice and have it keep working, including for automatic background backups. You can also choose between internal storage and an SD card for the live database. This brings Android in line with the custom-folder backup support already on other platforms.

### "Cavern" dive type

**Cavern** is now a built-in dive type you can select and filter by, and it's recognized when importing from other apps that use that label.

## 🐞 Bug fixes

### Bluetooth dive computers identified and downloaded correctly

Several BLE models were either mislabeled or wouldn't finish downloading:

- **OSTC nano** (and the wider hw_ostc3 family) now downloads successfully over Bluetooth — it previously connected but errored out during transfer.
- **Scubapro G2 HUD** is now identified as itself, instead of being mislabeled an Aladin Sport Matrix.
- **Halcyon Symbios Handset** is now labeled Handset rather than HUD (the two share a connection but are different products).
- **Halcyon Symbios** download timeouts are fixed by sending commands on the correct Bluetooth characteristic on every platform (the read/write direction was backwards).

### Cloud sync no longer runs out of memory on large libraries

Syncing a large dive library could crash with an out-of-memory error because the entire sync baseline was loaded into RAM at once. The baseline is now streamed to a temporary file and applied in batches, with a checksum verifying integrity. The result is proven byte-for-byte identical to the old in-memory path. Reconciling sync data now uses on the order of tens of megabytes of memory instead of potentially gigabytes.

### S3 buckets with a period in the name

Cloud sync to an S3-compatible bucket whose name contains a dot (for example `my.dive.bucket`) failed its connection test and sync because of how TLS certificates match such hostnames. Submersion now automatically uses path-style addressing for dotted bucket names over HTTPS, and saved configurations self-heal.

## 🛠️ Under the hood

- The CI/CD pipeline was parallelized and now shards tests by file, cutting build wall-clock time roughly from ~49 minutes to ~15–18 minutes — faster turnaround on fixes.
- Extensive new automated test coverage accompanies this release (over 20,000 lines added), particularly around backups, sync, dive-site fields, the profile chart, and rebreather imports.
