# Gas Timeline Default Visibility Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an appearance setting that controls whether the gas-usage timeline strip (the "Gases" line below the dive profile chart) is shown by default, defaulting to visible.

**Architecture:** Mirror the existing `defaultShowGasSwitchMarkers` setting end-to-end: a new persisted boolean column on the `DiverSettings` Drift table, threaded through `AppSettings`, the settings notifier, and `DiverSettingsRepository`, then consumed in `ProfileLegend.build()` (which already hydrates every other legend toggle from settings) and exposed as a `SwitchListTile` in the Dive Profile appearance section.

**Tech Stack:** Flutter, Drift ORM (codegen via build_runner), Riverpod, Flutter gen-l10n localization.

**Branch:** Work directly on `main` (per maintainer instruction). Commit after each task.

**Conventions:**
- No `Co-Authored-By` trailer in commit messages.
- Run specific test files (not whole directories) to avoid timeouts.
- `dart format .` before finishing.
